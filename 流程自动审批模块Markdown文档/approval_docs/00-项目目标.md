# 流程自动审批模块项目目标

## 1. 项目背景

本项目用于开发企业内部流程自动审批功能，当前第一阶段主要服务于非标自动化设备公司内部的两个核心业务流程：

1. ECN 变更审批
2. 新项目下单审批

系统目标是将原本依赖人工口头沟通、企业微信消息、Excel 记录的审批流程，转化为标准化、可追踪、可查询、可扩展的系统流程。

## 2. 核心目标

本模块不是让系统或 AI 直接代替人员判断是否批准，而是实现：

- 自动匹配审批流程
- 自动生成审批节点
- 自动流转给对应审批人
- 自动记录审批日志
- 自动维护审批状态
- 自动汇总待办、已办、抄送记录

## 3. 第一阶段功能范围

第一阶段只开发两个审批功能：

### 3.1 ECN 变更审批

用于设计变更、图纸变更、BOM 变更、物料替代、工艺调整等工程变更场景。

基础流程：

```text
申请人提交 ECN 变更申请
    ↓
研发负责人审批
    ↓
审批通过后抄送项目经理
    ↓
流程结束
```

### 3.2 新项目下单审批

用于新项目启动前的内部审批，确认项目资料、技术风险、交期风险、商务风险是否满足正式下单条件。

基础流程：

```text
申请人提交新项目下单申请
    ↓
项目经理审批
    ↓
研发负责人审批
    ↓
总经理审批
    ↓
审批通过后流程结束
```

## 4. 设计原则

### 4.1 通用审批引擎

ECN 变更审批和新项目下单审批不要各自开发独立审批逻辑，而是共用一套审批引擎。

推荐结构：

```text
业务单据
  ↓
审批实例
  ↓
审批节点
  ↓
审批日志
```

以后新增图纸审批、BOM 审批、采购审批、文件发布审批时，只需要新增业务类型和审批模板，不需要重写审批系统。

### 4.2 先简单，后扩展

第一阶段只实现：

- 串行审批
- 单人审批
- 固定审批流程
- 通过
- 驳回
- 退回修改
- 撤回
- 审批日志
- 我的待办
- 我发起的审批
- 抄送我的审批

暂时不实现：

- 并行审批
- 会签
- 或签
- 条件审批
- 流程设计器
- 企业微信通知
- 移动端审批

## 5. 最终交付目标

第一阶段完成后，系统应具备以下能力：

1. 用户可以创建 ECN 变更申请。
2. 用户可以创建新项目下单申请。
3. 用户可以提交审批。
4. 系统可以自动生成审批流程。
5. 审批人可以查看待办并审批。
6. 申请人可以查看审批进度。
7. 抄送人可以查看审批结果。
8. 所有审批动作都有日志记录。
9. 管理员可以配置基础审批流程。
10. 后续可以扩展到 PDM、BOM、图纸、采购等审批业务。
