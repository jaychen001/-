# 流程自动审批模块文档

本文件夹包含企业内部流程自动审批模块的开发文档。

当前第一阶段功能：

1. ECN 变更审批
2. 新项目下单审批

## 文档目录

| 文件 | 说明 |
|---|---|
| 00-项目目标.md | 项目背景、目标、范围 |
| 01-需求说明书.md | 业务需求和功能说明 |
| 02-审批流程设计.md | 审批状态、流程流转、节点规则 |
| 03-数据库设计.md | 数据库表结构和字段说明 |
| 04-API接口设计.md | 后端接口设计 |
| 05-前端页面设计.md | 页面结构和交互说明 |
| 06-后端模块设计.md | 后端模块划分和伪代码 |
| 07-开发任务清单.md | 开发任务、测试任务、验收标准 |
| 08-Codex工作规则.md | 给 Codex 使用的开发规则 |

## 推荐使用方式

把整个 docs 文件夹放到项目根目录下，然后对 Codex 说：

```text
请阅读 docs 目录下的 Markdown 文档，先不要写代码。
请先理解流程自动审批模块的业务目标、数据库设计、API 设计、审批流程设计和开发任务清单。
当前第一阶段只实现 ECN 变更审批和新项目下单审批。
请先输出你对项目的理解、推荐开发顺序、需要新增或修改的文件列表，然后再等待我确认。
```

## 第一阶段开发重点

第一阶段不要追求复杂流程设计器，优先完成：

- 串行审批
- 单人审批
- 审批通过
- 审批驳回
- 退回修改
- 审批日志
- 我的待审批
- 我发起的审批
- 抄送我的审批
- ECN 变更审批
- 新项目下单审批
