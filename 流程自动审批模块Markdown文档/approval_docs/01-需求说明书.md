# 流程自动审批模块需求说明书

## 1. 功能概述

本模块用于实现企业内部流程审批，第一阶段包含两个业务功能：

1. ECN 变更审批
2. 新项目下单审批

两个业务功能共用一套通用审批能力。

## 2. 用户角色

### 2.1 申请人

申请人可以：

- 创建审批单
- 保存草稿
- 提交审批
- 查看自己发起的审批
- 查看审批进度
- 被退回后修改并重新提交
- 在审批未开始或允许撤回时撤回审批

### 2.2 审批人

审批人可以：

- 查看待自己审批的流程
- 审批通过
- 审批驳回
- 退回修改
- 填写审批意见
- 查看审批历史

### 2.3 抄送人

抄送人可以：

- 查看抄送给自己的审批
- 查看审批结果
- 查看审批日志

抄送人不能执行审批操作。

### 2.4 管理员

管理员可以：

- 查看全部审批单
- 配置审批模板
- 配置审批节点
- 配置审批人
- 配置抄送人
- 启用或停用审批模板

## 3. 审批状态

### 3.1 审批单主状态

| 状态 | 编码 | 说明 |
|---|---|---|
| 草稿 | draft | 申请人创建后未提交 |
| 审批中 | pending | 已提交，正在审批 |
| 已通过 | approved | 所有审批节点已通过 |
| 已驳回 | rejected | 任意审批节点驳回 |
| 退回修改 | returned | 审批人要求申请人修改 |
| 已撤回 | withdrawn | 申请人主动撤回 |

### 3.2 审批节点状态

| 状态 | 编码 | 说明 |
|---|---|---|
| 未开始 | waiting | 节点尚未轮到审批 |
| 待审批 | pending | 当前节点等待审批人处理 |
| 已通过 | approved | 当前节点审批通过 |
| 已驳回 | rejected | 当前节点审批驳回 |
| 已跳过 | skipped | 当前节点被系统跳过 |

## 4. 功能一：ECN 变更审批

### 4.1 业务说明

ECN 变更审批用于处理工程变更，包括但不限于：

- 图纸变更
- BOM 变更
- 物料替代
- 工艺调整
- 设计方案调整
- 供应商或品牌替换
- 已发图纸版本更新

### 4.2 ECN 表单字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| ECN 编号 | string | 是 | 系统自动生成或手动输入 |
| 项目名称 | string | 是 | 所属项目 |
| 产品名称 | string | 否 | 涉及产品或设备 |
| 变更类型 | enum | 是 | 图纸/BOM/物料/工艺/其他 |
| 变更原因 | text | 是 | 为什么要变更 |
| 变更内容 | text | 是 | 具体变更内容 |
| 影响范围 | text | 是 | 影响哪些模块、图纸、物料 |
| 涉及图纸 | text/json | 否 | 图纸编号或附件 |
| 涉及 BOM | text/json | 否 | BOM 编号或附件 |
| 涉及物料 | text/json | 否 | 物料清单 |
| 是否影响成本 | boolean | 是 | 是/否 |
| 是否影响交期 | boolean | 是 | 是/否 |
| 是否影响库存 | boolean | 是 | 是/否 |
| 附件 | file[] | 否 | 图纸、说明、截图等 |
| 申请人 | user_id | 是 | 当前登录用户 |
| 申请时间 | datetime | 是 | 系统生成 |

### 4.3 ECN 审批流程

```text
申请人提交
    ↓
研发负责人审批
    ↓
通过后抄送项目经理
    ↓
流程结束
```

### 4.4 ECN 操作规则

1. 草稿状态允许申请人编辑。
2. 审批中状态不允许申请人直接修改。
3. 只有研发负责人可以审批。
4. 审批通过后 ECN 状态变为已通过。
5. 驳回后流程终止。
6. 退回修改后申请人可以修改并重新提交。
7. 所有操作必须写入审批日志。

## 5. 功能二：新项目下单审批

### 5.1 业务说明

新项目下单审批用于项目正式启动前的内部确认，避免资料不全、技术风险不清、交期无法满足、商务条件不明确等问题。

### 5.2 新项目下单表单字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| 项目编号 | string | 是 | 系统生成或手动输入 |
| 项目名称 | string | 是 | 项目名称 |
| 客户名称 | string | 是 | 客户名称 |
| 产品/设备名称 | string | 是 | 设备或产品名称 |
| 项目类型 | enum | 是 | 标准/非标/改造/打样 |
| 订单金额 | decimal | 否 | 项目金额 |
| 预计交期 | date | 是 | 客户要求交期 |
| 付款方式 | string | 否 | 付款条件 |
| 技术要求 | text | 是 | 项目核心技术要求 |
| 是否非标项目 | boolean | 是 | 是/否 |
| 是否需要打样 | boolean | 是 | 是/否 |
| 是否需要采购长周期物料 | boolean | 是 | 是/否 |
| 项目风险说明 | text | 否 | 风险描述 |
| 客户资料附件 | file[] | 否 | 客户需求资料 |
| 技术协议附件 | file[] | 否 | 技术协议 |
| 合同附件 | file[] | 否 | 合同或订单 |
| 申请人 | user_id | 是 | 当前登录用户 |
| 申请时间 | datetime | 是 | 系统生成 |

### 5.3 新项目下单审批流程

```text
申请人提交
    ↓
项目经理审批
    ↓
研发负责人审批
    ↓
总经理审批
    ↓
流程结束
```

### 5.4 节点职责

#### 项目经理审批

重点确认：

- 项目资料是否完整
- 客户需求是否明确
- 交期是否合理
- 项目风险是否已说明

#### 研发负责人审批

重点确认：

- 技术方案是否可行
- 研发资源是否满足
- 是否存在重大技术风险
- 是否需要提前验证或打样

#### 总经理审批

重点确认：

- 项目是否值得执行
- 商务风险是否可接受
- 资源投入是否合理
- 是否批准正式下单

## 6. 通用审批能力

系统需要支持以下通用能力：

1. 创建草稿
2. 提交审批
3. 自动匹配审批模板
4. 自动生成审批实例
5. 自动生成审批节点
6. 自动激活第一个审批节点
7. 审批通过
8. 审批驳回
9. 退回修改
10. 撤回审批
11. 自动流转下一节点
12. 审批完成
13. 抄送人员
14. 审批日志

## 7. 异常规则

1. 未配置审批模板时，不允许提交审批。
2. 审批模板未启用时，不允许提交审批。
3. 审批节点未配置审批人时，不允许提交审批。
4. 当前用户不是当前节点审批人时，不允许审批。
5. 已通过、已驳回、已撤回的审批单不能再次审批。
6. 审批中状态不允许直接修改业务单据。
7. 退回修改后，申请人可以修改并重新提交。
