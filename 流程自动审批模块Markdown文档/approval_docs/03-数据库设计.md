# 数据库设计

## 1. 设计说明

本数据库设计用于支持企业内部流程自动审批模块。

第一阶段支持：

1. ECN 变更审批
2. 新项目下单审批

数据库设计分为两部分：

1. 通用审批引擎表
2. 具体业务单据表

## 2. 通用审批引擎表

### 2.1 approval_templates：审批模板表

用于定义不同业务类型对应的审批流程模板。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| name | varchar(100) | 模板名称 |
| code | varchar(100) | 模板编码 |
| business_type | varchar(100) | 业务类型 |
| description | text | 模板说明 |
| is_active | boolean | 是否启用 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

示例：

| name | code | business_type |
|---|---|---|
| ECN 变更审批 | ECN_APPROVAL | ECN_APPROVAL |
| 新项目下单审批 | PROJECT_ORDER_APPROVAL | PROJECT_ORDER_APPROVAL |

### 2.2 approval_template_nodes：审批模板节点表

用于定义模板中的审批节点。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| template_id | bigint | 审批模板 ID |
| node_name | varchar(100) | 节点名称 |
| node_type | varchar(50) | 节点类型 |
| approver_type | varchar(50) | 审批人类型 |
| approver_id | bigint | 固定审批人 ID |
| role_code | varchar(100) | 审批角色编码 |
| sort_order | int | 节点顺序 |
| is_required | boolean | 是否必审 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

说明：

- approver_type 可以是 USER、ROLE。
- 第一阶段建议优先支持 ROLE。
- 例如 RND_MANAGER、PROJECT_MANAGER、GENERAL_MANAGER。

### 2.3 approval_instances：审批实例表

用于保存每一次具体发起的审批流程。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| template_id | bigint | 审批模板 ID |
| business_type | varchar(100) | 业务类型 |
| business_id | bigint | 业务单据 ID |
| title | varchar(200) | 审批标题 |
| applicant_id | bigint | 申请人 ID |
| status | varchar(50) | 审批状态 |
| current_node_id | bigint | 当前审批节点 ID |
| submitted_at | datetime | 提交时间 |
| completed_at | datetime | 完成时间 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

status 可选值：

```text
draft
pending
approved
rejected
returned
withdrawn
```

### 2.4 approval_instance_nodes：审批实例节点表

用于保存某一次审批实例中的具体审批节点。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| instance_id | bigint | 审批实例 ID |
| template_node_id | bigint | 模板节点 ID |
| node_name | varchar(100) | 节点名称 |
| approver_id | bigint | 审批人 ID |
| status | varchar(50) | 节点状态 |
| sort_order | int | 节点顺序 |
| approved_at | datetime | 审批时间 |
| comment | text | 审批意见 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

status 可选值：

```text
waiting
pending
approved
rejected
skipped
```

### 2.5 approval_logs：审批日志表

用于保存审批过程中的所有操作记录。

| to_status | varchar(50) | 操作后状态 |
| created_at | datetime | 创建时间 |

action 可选值：

```text

submit
approve
reject
return
withdraw
auto_next
complete

```

### 2.6 approval_cc_users：审批抄送人表

用于保存审批抄送记录。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |

| read_status | varchar(50) | 已读状态 |
| read_at | datetime | 阅读时间 |

read_status 可选值：

```text

```

## 4. 用户与角色表建议

如果系统已有用户权限模块，可以复用已有表。

如果暂时没有，建议建立最小用户表和角色表。

### 4.1 users：用户表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint | 主键 |
| username | varchar(100) | 用户名 |

| email | varchar(200) | 邮箱 |

| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

## 6. 数据一致性要求

1. 提交审批时，业务单据状态和审批实例状态必须同时更新。
2. 审批通过时，节点状态、实例状态、业务状态必须在同一事务中更新。
3. 审批驳回时，节点状态、实例状态、业务状态必须在同一事务中更新。
4. 所有审批动作必须写入 approval_logs。
5. 重新提交时，不建议覆盖旧审批实例，应创建新的审批实例。
