# 工程审批软件项目章程

## 核心原则

### 1. 先规格，后实现
每个新增功能先沉淀业务目标、用户路径、验收标准和数据约束，再进入代码实现。

### 2. 面向非标自动化工程场景
界面、字段和流程必须体现制造业工程管理特征，优先服务 ECN 变更、新项目下单、项目进度管控和项目信息变更。

### 3. 小改动直接记录，大改动走审批
普通节点进度信息允许直接编辑，但必须留下操作记录；关键项目信息变化必须走项目信息变更审批；图纸、BOM、设计方案、采购替代等工程变化必须走 ECN。

### 4. 自动化能力可见
原型和正式系统都要清楚展示自动编号、自动流转、自动抄送、自动提醒、自动归档、自动创建项目节点和自动生成变更记录。

### 5. 纯前端 Demo 不依赖后端
当前阶段前端原型全部使用模拟数据，打开 `frontend-prototype/index.html` 即可演示，不引入复杂框架。

## 当前技术约束

- 主实现文件：`frontend-prototype/index.html`、`frontend-prototype/styles.css`、`frontend-prototype/app.js`
- 不引入后端接口
- 不引入复杂前端框架
- 不展示资金类字段和相关功能
- 桌面端演示优先，避免文字、按钮、表格重叠

## 验收门槛

- 页面无 JavaScript 语法错误
- 导航、按钮、弹窗、详情页可演示
- ECN、新项目下单、项目信息变更、项目进度管控形成闭环
- 新项目审批通过后可导入项目进度管控并自动生成节点
- 项目详情可编辑普通节点并生成操作记录
- 关键变更可跳转项目信息变更审批
- 工程设计变化可跳转 ECN 变更审批
